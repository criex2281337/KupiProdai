Руководство пользователя
Программный продукт «Купи-Продай»
1. Введение
1.1. Область применения
Программный продукт «Купи-Продай» предназначен для автоматизации учёта и управления торговой деятельностью предприятия. Система используется для хранения, редактирования и анализа данных о товарах, контрагентах, ценах, а также для управления движением товаров на складах.
1.2. Краткое описание возможностей
Система предоставляет следующие функциональные возможности:
1) Управление номенклатурой товаров (добавление, редактирование, удаление)
2) Учёт движения товаров (поступление и реализация)
3) Управление ценами на товары
4) Контроль остатков товаров на складах
5) Формирование отчетов по остаткам и прайс-листов
6) Экспорт данных в формат JSON
7) Поиск и фильтрация товаров
8) Автоматический расчет сумм и остатков
1.3. Уровень подготовки пользователя
Для работы с программой достаточно начальных навыков пользования компьютером и базовых знаний работы с приложениями Windows.
1.4. Перечень эксплуатационной документации
1) Руководство пользователя (данный документ)
2) Описание структуры базы данных в формате JSON
3) Техническое задание на разработку системы
2. Назначение и условия применения
2.1. Виды деятельности, функции
Программа выполняет следующие функции:
1) Учёт данных о номенклатуре товаров, их категориях и единицах измерения
2) Ведение информации о движении товаров (поступление и реализация)
3) Управление остатками товаров на складах
4) Контроль цен и автоматический их расчет
5) Формирование аналитических отчетов
6) Экспорт и импорт данных
2.2. Программные и адаптивные требования
Для корректной работы программы необходимы:
1) Операционная система Windows 7 и выше
2) Установленный .NET Framework версии 4.5 или новее
3) Разрешение экрана не ниже 1024×768
3. Подготовка к работе
3.1. Состав дистрибутива
В состав программного продукта входят:
1) Файл `KupiProdai.exe` — исполняемый модуль программы
2) Папка `Data` — содержит файлы базы данных в формате JSON:
   - `products.json` — данные о товарах
   - `movements.json` — данные о движении товаров
   - `suppliers.json` — данные о поставщиках
   - `warehouses.json` — данные о складах
3) Руководство пользователя
3.2. Запуск системы
1) Запустить файл `KupiProdai.exe`
2) После запуска появится главное окно программы с двумя вкладками:
   - «Товары» — для управления номенклатурой
   - «Движение товаров» — для учета поступлений и реализаций
3.3. Проверка работоспособности
После запуска программы проверить:
1) Корректность открытия главного окна программы
2) Доступность всех кнопок управления
3) Возможность добавления и редактирования товаров
4) Корректность отображения таблиц с данными
4. Описание операций
4.1. Операция: Управление товарами
4.1.1. Наименование операции
Добавление, редактирование и удаление товаров в системе.
4.1.2. Условия выполнения
Открыта вкладка «Товары» в главном окне программы.
4.1.3. Подготовительные действия
1. Перейти на вкладку «Товары»
2. Заполнить поля формы товара
4.1.4. Основные действия
1. Для добавления товара:
   - Ввести наименование товара
   - Указать код товара
   - Выбрать категорию («Товар» или «Услуга»)
   - Выбрать единицу измерения
   - Указать цену и начальное количество
   - Нажать кнопку «Добавить»
2. Для редактирования товара:
   - Выбрать товар в таблице (кликнуть по строке)
   - Изменить необходимые поля в форме
   - Нажать кнопку «Обновить»
3. Для удаления товара:
   - Выбрать товар в таблице
   - Нажать кнопку «Удалить»
   - Подтвердить операцию
4.1.5. Заключительные действия
После выполнения операции данные автоматически сохраняются в файл `products.json`.
4.1.6. Ресурсы, расходуемые на операцию
Используется файл базы данных `products.json`.
4.2. Операция: Учёт движения товаров
4.2.1. Наименование операции
Добавление записей о поступлении или реализации товаров.
4.2.2. Условия выполнения
Открыта вкладка «Движение товаров» в главном окне программы.
4.2.3. Подготовительные действия
1. Перейти на вкладку «Движение товаров»
2. Заполнить поля формы движения
4.2.4. Основные действия
1. Для добавления движения:
   - Ввести название товара
   - Выбрать тип операции («Поступление» или «Реализация»)
   - Выбрать склад
   - Указать количество и цену
   - Установить дату операции
   - Нажать кнопку «Добавить движение»
4.2.5. Заключительные действия
После добавления движения:
1. Запись добавляется в таблицу движений
2. Автоматически пересчитывается остаток товара на складе
3. Данные сохраняются в файл `movements.json`
4.2.6. Ресурсы, расходуемые на операцию
Используются файлы базы данных `movements.json` и `products.json`.
4.3. Операция: Поиск товаров
4.3.1. Наименование операции
Поиск товаров по различным критериям.
4.3.2. Условия выполнения
Открыта вкладка «Товары» в главном окне программы.
4.3.3. Подготовительные действия
1. Перейти на вкладку «Товары»
2. Найти поле «Поиск товаров...» в верхней части окна
4.3.4. Основные действия
1. Ввести текст для поиска в поле «Поиск товаров...»
2. Система автоматически отфильтрует товары по:
   - Наименованию
   - Коду товара
   - Категории

4.3.5. Заключительные действия
В таблице отображаются только товары, соответствующие критериям поиска.
4.3.6. Ресурсы, расходуемые на операцию
Используются данные из файла `products.json`.
4.4. Операция: Экспорт данных
4.4.1. Наименование операции
Экспорт данных о товарах в файл JSON.
4.4.2. Условия выполнения
Открыта вкладка «Товары» в главном окне программы.
4.4.3. Подготовительные действия
1. Перейти на вкладку «Товары»
2. Убедиться, что в таблице есть данные для экспорта
4.4.4. Основные действия
1. Нажать кнопку «Экспорт»
2. В открывшемся диалоговом окне выбрать папку для сохранения
3. Указать имя файла (по умолчанию: `товары_дата_время.json`)
4. Нажать кнопку «Сохранить»
4.4.5. Заключительные действия
Создается файл JSON с данными о всех товарах в системе.
4.4.6. Ресурсы, расходуемые на операцию
Используются данные из файла `products.json`.
4.5. Операция: Формирование отчета
4.5.1. Наименование операции
Формирование отчета о товарах с низким остатком.
4.5.2. Условия выполнения
Открыта вкладка «Товары» в главном окне программы.
4.5.3. Подготовительные действия
1. Перейти на вкладку «Товары»
2. Нажать кнопку «Отчет»
4.5.4. Основные действия
1. Система автоматически анализирует остатки товаров
2. Формируется список товаров с количеством менее 10 единиц
3. Отчет отображается в диалоговом окне
4.5.5. Заключительные действия
Пользователь получает информацию о товарах, которые необходимо пополнить.
4.5.6. Ресурсы, расходуемые на операцию
Используются данные из файла `products.json`.
4.6. Операция: Автоматический расчет
4.6.1. Наименование операции
Автоматический расчет сумм и остатков.
4.6.2. Условия выполнения
Добавление или изменение данных о движении товаров.
4.6.3. Подготовительные действия
Нет специальных подготовительных действий.
4.6.4. Основные действия
1. При добавлении движения товара:
   - Автоматически рассчитывается сумма (количество × цена)
   - При поступлении: остаток товара увеличивается
   - При реализации: остаток товара уменьшается
   - Контролируется, чтобы остаток не стал отрицательным
2. При редактировании товара:
   - Пересчитываются связанные движения
4.6.5. Заключительные действия
Данные автоматически обновляются во всех связанных таблицах.
4.6.6. Ресурсы, расходуемые на операцию
Используются все связанные файлы базы данных.

6. Рекомендации по освоению
6.1. Начало работы
1. Первое использование: При первом запуске программы папка `Data` будет создана автоматически с пустыми файлами JSON.
2. Добавление товаров: Начните с добавления основных товаров через вкладку «Товары».
3. Создание складов: Убедитесь, что в системе есть хотя бы один склад для учета остатков.


6.2. Регулярная работа
1. Ежедневные операции: Используйте вкладку «Движение товаров» для ежедневного учета поступлений и реализаций.
2. Контроль остатков: Регулярно проверяйте остатки товаров через таблицу на вкладке «Товары» или с помощью кнопки «Отчет».
3. Резервное копирование: Периодически экспортируйте данные через кнопку «Экспорт» для создания резервных копий.
6.3. Оптимизация работы
1. Использование поиска: Для быстрого нахождения товаров используйте поле поиска на вкладке «Товары».
2. Автозаполнение: При добавлении движения товара система автоматически подставляет цену из последней установленной цены.
3. Валидация данных: Программа автоматически проверяет корректность вводимых данных и предотвращает ошибки.
6.4. Безопасность и сохранность данных
1. Автоматическое сохранение: Все изменения сохраняются автоматически при каждом действии.
2. Целостность данных: Система обеспечивает целостность данных при любых операциях.
3. Восстановление: При повреждении файлов JSON можно восстановить данные из экспортированных файлов.
6.5. Расширенные возможности
1. Аналитика: Используйте отчеты для анализа товаров с низким остатком.
2. Интеграция: Экспортированные файлы JSON можно использовать в других системах учета.
3. Масштабирование: Система поддерживает неограниченное количество товаров, складов и операций.
